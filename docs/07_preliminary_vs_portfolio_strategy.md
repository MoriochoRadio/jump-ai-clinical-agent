# Preliminary Round And Portfolio Strategy

## Key Question

Do we only prepare the preliminary proposal and wait for finalist selection, or do we also build parts of the project beforehand?

## Competition Structure

The preliminary round is proposal-centered. The main required output at this stage is the official proposal form.

The final round, if selected, requires actual agent development and final outputs such as:

- demo video,
- technical/research report,
- service deployment link,
- presentation materials.

## Practical Answer

We do not have to wait passively.

Even though the competition only requires a proposal in the preliminary round, building selected parts early is useful because:

- it makes the proposal more credible,
- it produces a GitHub portfolio even if the team is not selected,
- it helps verify whether the idea is technically realistic,
- it gives us concrete screenshots, diagrams, and experiments later.

However, we should not build the full product too early.

## Recommended Strategy

Use a two-track approach.

### Track A: Submission Track

Primary goal:

- write the best possible preliminary proposal.

Outputs:

- one-page concept note,
- proposal outline,
- rubric mapping,
- final 10-page proposal draft.

### Track B: Portfolio Validation Track

Secondary goal:

- build small, safe proof-of-concept pieces that support the proposal.

Allowed early outputs:

- source/API feasibility notes,
- sample workflow diagram,
- dummy clinical trial review checklist,
- one or two synthetic test scenarios,
- very small CLI or notebook experiment.

Avoid for now:

- full web app,
- expensive integrations,
- broad multi-agent framework,
- claims of medical/regulatory validation,
- large-scale model training.

## Why This Is Safer

If selected for the final round:

- we already have a research base and small experiments,
- final development starts faster.

If not selected:

- the GitHub repository still shows a careful Medical IT project,
- the concept can become an independent portfolio project,
- the work can be reused for another healthcare AI or hospital IT project.

## Current Recommendation

Proceed with:

1. one-page concept note for Field 3,
2. then small feasibility research,
3. then proposal outline,
4. only after that, a minimal prototype experiment.

This keeps the pace conservative while still making sure the work is useful even outside the competition.
