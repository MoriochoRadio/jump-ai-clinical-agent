# Project Guidance

## Goal

Build a portfolio-quality project for the 4th JUMP AI drug development challenge, with a strong Medical IT and hospital information system angle.

## Working Principles

- Keep every major decision documented.
- Prefer practical healthcare workflows over flashy but unrealistic model claims.
- Separate facts from assumptions.
- Track sources for medical, regulatory, and competition claims.
- Do not store API keys, credentials, downloaded private competition files, or personal data in the repo.

## Preferred Direction

Prioritize agent ideas connected to:

- Clinical trial protocol design
- Regulatory and guideline checks
- Hospital data readiness
- EMR/HIS integration assumptions
- Evidence retrieval and traceable reporting

Avoid starting with:

- Claims of training a new foundation model
- Unsupported medical diagnosis claims
- Pure molecule generation without domain validation

## Documentation Style

Write in Korean when the material is for the user or proposal planning. Use English for code, file names, and concise technical labels where helpful.
